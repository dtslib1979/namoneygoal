# 남원골 / namoneygoal

> 시흥 부동산중개인용 AI 워키토키
> AI 리터러시를 "허세 소비재"로 파는 직업 운영 시스템

---

## 0. 한 줄 요약

남원골은 생산성 도구가 아니다.

> "나는 AI를 이렇게 쓴다"
> "나는 디지털 명함이 있다"
> "나는 내 블로그, 내 유튜브, 내 사무실이 있다"

를 자랑할 수 있게 만들어주는 **허세 소비재**다.

생산성은 보너스고, 구매 동기는 즉시 만족이다.

---

## 1. 정체성

남원골은:
- 강의가 아니다
- 컨설팅이 아니다
- SaaS가 아니다

**이건 직업용 라이프스타일 상품이다.**

중개인은 더 이상 "부동산 중개인"이 아니라:
- 내 디지털 명함이 있고
- 내 디지털 블로그가 있고
- 내 디지털 유튜브가 있고
- 내 디지털 사무실이 있고
- "나는 AI를 이렇게 쓴다"라고 말할 수 있는

**현대적 직업인 캐릭터**가 된다.

### 플랫폼 역할

| 플랫폼 | 역할 |
|--------|------|
| 유튜브 | CDN (저장소) |
| 네이버 | 본진 (유통) |
| AI | 말을 번역하는 엔진 |

중개인은 "유튜버"가 되는 게 아니라
**"말이 자동으로 기록되는 인간"**이 된다.

---

## 2. 구조 – 브랜치의 브랜치

남원골은 DTSLIB Branch OS의 **첫 번째 버티컬 브랜치**다.

### 전체 구조

```
DTSLIB Branch HQ (본사)
    │
    ├── KOOSY
    ├── GOHSY
    ├── ARTREW
    ├── PAPAFLY
    ├── BUCKLEY
    ├── NAMONEYGOAL ◀── 브랜치
    │       │
    │       ├── slot01 ◀── 브랜치의 브랜치
    │       ├── slot02
    │       ├── ...
    │       └── slot12
    │
    └── ...
```

### 레포 구조

```
namoneygoal/
 ├─ index.html        (남원골 HQ)
 ├─ slots/
 │   ├─ slot01 ~ slot12
 ├─ modules/
 │   ├─ card.html     (내 디지털 명함)
 │   ├─ blog.html     (내 디지털 블로그)
 │   ├─ video.html    (내 디지털 유튜브)
 │   ├─ walkie.html   (AI 워키토키)
 │   └─ pr.js         (PR 버튼)
 └─ console/
     ├─ requests.html (요청 로그)
     └─ billing.html  (정산 보드)
```

### 슬롯 구조

- **12명 한정**
- 친구 1명 무료 (앵커)
- 유료 11명

겉으로는 "각자의 사이트", 실제로는 **단일 관제 시스템**.

## 3. 허세 포트폴리오

남원골에 들어오면 자동으로 생기는 것:

| 이름 | 실체 |
|------|------|
| 내 디지털 명함 | card.html |
| 내 디지털 블로그 | blog.html |
| 내 디지털 유튜브 | video.html |
| 내 디지털 사무실 | slot.html |
| 내 AI 워키토키 | walkie.html |
| "남원골 멤버" | 소속감 |

**이건 스타터 팩이다.**

### 비유

```
헬스장 등록 → 운동하러 ❌
헬스장 등록 → "나 헬스장 다녀" ⭕

남원골 등록 → AI 공부하려고 ❌
남원골 등록 → "나 AI 쓴다" ⭕
```

이 차이가 구매를 만든다.

## 4. 유저 액션 (딱 이것만)

- "AI 써보실래요?" 같은 설명 없음

## 5. 운영자 역할

운영자는 강사가 아니다. **Operator(운영자)**다.

| 역할 | 내용 |
|------|------|
| 구조 설계 | 전체 시스템 아키텍처 |
| 캐릭터 번역 | 중개인 → 디지털 캐릭터 |
| 첫 글/멘트 세팅 | 초기 콘텐츠 제작 |
| HQ 운영 | 관제, 모니터링 |
| PR 요청 처리 | 시스템 호출 대응 |
| PD 모드 | 배치 콘텐츠 제작 |

### 세 가지 역할 동시 수행

```
DTSLIB 전체    → Architect (설계자)
남원골         → Operator (운영자)
슬롯 콘텐츠    → PD (제작자)
```

---

## 6. PR 시스템 연결

Branch HQ의 PR 요청 시스템이 그대로 연결된다.

```
중개인이 PR 버튼 누름
    ↓
자동 접수 (이름, 도메인, 요청 타입, 텍스트)
    ↓
진짜 회사 전산처럼 처리
    ↓
운영자가 대응
```

**"부탁"이 아니라 "시스템 호출".**

## 7. 가격 구조

대상: 이미 ChatGPT에 월 3만 쓰는 상위 필터층

### 기본 패키지: 50만 원 (입장비)

| 포함 항목 |
|-----------|
| 슬롯 생성 |
| 도메인 연결 |
| 남원골 키트 세팅 |
| 첫 글 / 첫 멘트 세팅 |
| 길드 등록 |

### 월 패키지: 10만 원

| 구성 | 금액 |
|------|------|
| ChatGPT 요금 | 본인 부담 3만 |
| 남원골 이용료 | 7만 (운영자 몫) |

| 포함 서비스 |
|-------------|
| 개인 슬롯 유지 |
| 남원골 HQ 관제 |
| PR 채널 사용 |
| 박씨 캡처 지원 |
| "AI 쓰는 중개인" 멤버십 |

### 요청형 부가 서비스 (변동 수익)

| 서비스 | 가격 | 설명 |
|--------|------|------|
| PWA/APK 제작 | 5만/건 | "내 앱" 느낌 완성 |
| 페이지 수정 | 1만/건 | 문구/디자인/구조 변경 |

### 중개인 체감

> "월 10만이면
> 내 디지털 명함,
> 내 블로그,
> 내 유튜브,
> 내 디지털 사무실,
> 그리고 'AI 쓰는 중개인' 캐릭터가 생긴다."

이건 도구 값이 아니라 **캐릭터 값**이다.

### 운영자 수익 구조

```
고정 수익: 11명 × 7만 = 77만/월
변동 수익: PWA + 수정 요청 = 20~30만/월
─────────────────────────────
바닥값: 약 100만/월
```

강의 없이, 콘텐츠 없이, **운영자로만** 발생하는 수익.

## 8. 심리 레이어

| 레이어 | 메시지 | 체감 시점 |
|--------|--------|-----------|
| 허세 소비 | "나 AI 쓴다" | 즉시 |
| 캐릭터 | "내 디지털 사무실 있다" | 즉시 |

## 10. 생산 전략

| 문제 | 해결 |
|------|------|
| "중개인이 안 하면?" | 운영자가 PD 모드로 대신 찍음 |
| "샘플이 없으면?" | 찍으면서 샘플이 됨 |
| "검증이 안 되면?" | 배치 작업으로 일괄 생산 |

**생산 병목 없음. 운영자가 직접 한다.**

---

## 11. 궤도 전략 연결

```
Year 0-1: 남원골 (시흥 부동산)
    ↓
검증되면 복제:
    - 다른 지역 부동산
    - 다른 업종 (보험, 학원, ...)
    ↓
각 버티컬이 브랜치가 되고
각 브랜치가 다시 슬롯을 갖는다
```

**남원골은 첫 번째 버티컬 브랜치의 파일럿이다.**
